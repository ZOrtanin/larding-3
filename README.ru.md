# Larding CMS 3.0.0

Larding CMS — это лёгкая CMS на базе Laravel для лендингов, небольших корпоративных сайтов и кастомных клиентских проектов.

Этот релиз включает веб-установщик, блочное управление страницей, сбор лидов, файловый менеджер, роли пользователей и административную панель, которую можно развернуть на стандартном PHP-хостинге.

## Статус релиза

`Larding 3.0.0` — первый публичный релиз проекта.

Этот пакет подготовлен как готовая к установке поставка:
- production `vendor` уже включён
- мастер установки уже включён
- Node.js на хостинге не требуется

## Основные возможности

- Веб-установщик по адресу `/install`
- Блочная структура лендинга
- Админ-панель для управления контентом
- Обработка лид-форм
- Файловый менеджер
- Пользовательские роли с поддержкой супер-администратора
- Стартовый seeded-контент для новой установки

## Системные требования

- PHP `8.2+`
- MySQL `8+` или совместимая MariaDB
- PHP-расширения:
  - `pdo`
  - `pdo_mysql`
  - `mbstring`
  - `openssl`
  - `fileinfo`

Установщику также нужны права на запись в:

- `storage/`
- `bootstrap/cache/`

## Установка

### 1. Загрузка файлов

Загрузите содержимое этого пакета на хостинг.

Если ваш хостинг позволяет менять `DocumentRoot`, укажите папку `public/`.

Если ваш хостинг не позволяет менять `DocumentRoot`, этот релиз уже подготовлен для запуска из корня сайта и включает:

- корневой `index.php`
- корневой `.htaccess`

То есть пакет можно размещать прямо в публичной веб-директории.

### 2. Подготовка окружения

Создайте `.env` из `.env.example`:

```bash
cp .env.example .env
```

Минимально задайте:

- `APP_ENV=production`
- `APP_DEBUG=false`
- `APP_INSTALLED=false`
- ваши параметры подключения `DB_*`

Не помечайте приложение как установленное до прохождения мастера установки.

### 3. Запуск установщика

Откройте сайт в браузере:

```text
https://your-domain/install
```

Сценарий установки:

1. Проверка окружения
2. Настройка базы данных
3. Инициализация базы
4. Создание первого администратора

После успешной установки приложение сохранит финальные параметры и пометит себя как установленное.

## Примечания по хостингу

- Node.js на production-хостинге не требуется
- фронтенд-ассеты уже включены в пакет
- этот билд рассчитан на стандартный shared hosting или простой VPS

## Структура проекта

- `app/`, `bootstrap/`, `config/`, `database/`, `resources/`, `routes/`:
  исходный код приложения
- `public/`:
  публичные ассеты
- `vendor/`:
  production PHP-зависимости
- `storage/`:
  runtime-файлы и логи
